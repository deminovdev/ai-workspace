# Агенты

В этой директории — агенты для конкретных задач. Каждый `.md` файл это агент с frontmatter `name`, `model`, `tools` и системным промптом.

## Маркетплейс

**Не пиши агента с нуля если он уже существует.**

→ [VoltAgent/awesome-claude-code-subagents](https://github.com/VoltAgent/awesome-claude-code-subagents) — 130+ агентов по категориям: backend, frontend, devops, security, data, testing и др.

Установка одной командой:
```bash
curl -fsSL https://raw.githubusercontent.com/VoltAgent/awesome-claude-code-subagents/main/install.sh \
  | bash -s -- <agent-name>
# Агент попадает в ~/.claude/agents/ (глобально) или .claude/agents/ (проект)
```

→ [hesreallyhim/awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code) — более широкий список: агенты, CLAUDE.md примеры, slash-команды, хуки.

## Как подбирать агента под задачу

Когда пользователь просит запустить агента — сначала проверь есть ли подходящий в маркетплейсе:

1. **Определи роль**: что агент должен делать — писать тесты, ревьюить код, работать с БД, деплоить?
2. **Поищи в маркетплейсе**: открой VoltAgent репо, найди по категории или ключевому слову
3. **Оцени tools**: агент должен иметь минимально необходимые инструменты — не больше
4. **Предложи пользователю**: покажи имя + что умеет + команду установки

Пример диалога:
> Пользователь: нужен агент для написания Go тестов
> Ты: В маркетплейсе есть `go-test-writer`. Установить?
> `curl ... | bash -s -- go-test-writer`

## Когда писать своего агента

- Задача специфична для проекта (знает структуру папок, соглашения)
- Нужны особые права — только на конкретную поддиректорию
- Комбинирует несколько ролей нестандартным образом

Шаблон:
```markdown
---
name: my-agent
description: Одна строка — когда запускать этого агента
model: sonnet
tools: Read, Grep, Glob, Edit, Write, Bash
---

Системный промпт. Что делает, чего не делает, в каком формате отвечает.
```
