# Loop Monitor

Наблюдатель за фоновыми агентами. Только читает — никогда не вмешивается в работу.

## Что проверяет

- **Прогресс по git** — есть ли новые коммиты или изменённые файлы за последние N минут
- **Активность файлов** — изменился ли рабочий каталог
- **Git log** — движется ли ветка вперёд

## Ограничения

Агент не видит внутреннее состояние другой сессии Claude. Он оценивает прогресс
**косвенно** — через изменения в файловой системе и git. Если агент думает
но ещё не пишет файлы — loop-monitor не отличит это от зависания.

Используй как дополнительный инструмент, не как абсолютный индикатор.

## Как использовать

```
Запусти loop-monitor: наблюдай за прогрессом в tmp/backend-refactor.
Проверяй каждые 10 минут. Сообщи если за 20 минут не будет новых коммитов или изменений файлов.
```

При отсутствии прогресса — сообщает в чат. Решение принимает человек.
