# Паттерн: Оркестратор

## Суть

Основной чат — точка координации. Код не пишется здесь. Задачи делегируются агентам, результаты сводятся вместе.

Агенты запускаются параллельно через `Agent` tool (subagents). Каждый видит свой контекст, работает независимо, возвращает результат в основной чат.

## Когда применять

- Задача затрагивает несколько сервисов или репозиториев
- Нужна параллельная работа: ресёрч + реализация + тест одновременно
- Задача слишком большая для одного контекста

## Когда НЕ применять

- Задача в одном сервисе → [Worktree Agent](worktree-agent.md)
- Простая правка в одном файле → работай напрямую
- Нет зависимостей между частями → каждую можно сделать отдельно

## Структура

```
Основной чат (оркестратор)
  ├── researcher  →  анализ, поиск решения
  ├── architect   →  оценка подхода, trade-offs
  └── backend     →  реализация (worktree)
      └── frontend →  реализация (worktree)
```

## Пример

Задача: добавить авторизацию через OAuth.

```
1. Запусти researcher: как обычно реализуют OAuth с нашим стеком (Go + React)?

2. [researcher завершил]
   Запусти architect: оцени подход из результата researcher — есть ли проблемы с нашей архитектурой?

3. [architect завершил]
   Запусти backend-агента в worktree: реализовать OAuth endpoints по схеме [вставь результат architect]
   Запусти frontend-агента в worktree: добавить OAuth flow на UI

4. [оба завершили]
   Запусти reviewer: проверь изменения в tmp/backend-oauth и tmp/frontend-oauth
```

## Правила оркестратора

**Не пиши код в основном чате.** Только координация — читай результаты, принимай решения, делегируй дальше.

**Запускай параллельно** всё что не зависит друг от друга. `researcher` и проверка окружения — одновременно.

**Передавай контекст явно.** Агент не читал предыдущие сообщения. В промпте опиши: что нужно сделать, где, какие ограничения, какой результат ожидаешь.

**Синтезируй сам.** Если два агента нашли противоречащие решения — ты принимаешь решение, не агент.

## Ограничения

Агент (subagent) работает в изолированном контексте. Он не знает что сделал другой агент, если ты не передал это явно в промпте.

Агенты не общаются напрямую — только через основной чат.

## Настройка workspace

AGENTS.md должен содержать карту проектов — что в каком `tmp/` репо, структуру директорий, как запустить. Без этого агентам нечего читать при старте.

```markdown
## Твои проекты

### backend — tmp/backend/
Go сервис. Запуск: make dev. Код: internal/

### frontend — tmp/frontend/
React. Запуск: npm run dev. Код: src/
```
