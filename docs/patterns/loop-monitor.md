# Паттерн: Loop Monitor

## Суть

Лёгкий watchdog-агент, который наблюдает за фоновыми агентами через файловую систему и git. Не вмешивается в работу — только наблюдает и сигналит.

## Проблема

Фоновый агент может зависнуть:
- Ждёт ввода который никто не даёт
- Зациклился на одном файле
- Упёрся в permission-блок и не сообщил об этом
- Тихо умер без результата

Без мониторинга это не заметно пока не проверишь вручную.

## Что loop-monitor МОЖЕТ

- Следить за временем последнего изменения файлов в `tmp/<worktree>/`
- Проверять `git log` — был ли новый коммит в последние N минут
- Читать логи из `modules/hooks/.stop.log` — завершились ли сессии
- Наблюдать за `~/.claude/projects/**/*.jsonl` — активность сессий по файлу

## Что loop-monitor НЕ МОЖЕТ

- Видеть что происходит внутри Claude-сессии в реальном времени
- Определить зависание без внешних признаков (изменение файлов, коммиты)
- Остановить другой агент напрямую
- Подключиться к чужой сессии

## Когда использовать

```
Запусти loop-monitor: следи за tmp/backend-auth-oauth,
сообщи если 20 минут не будет новых коммитов или изменений файлов
```

## Пример мониторинга

```bash
# Что делает loop-monitor внутри:
while true; do
  last_change=$(find tmp/backend-auth-oauth -newer /tmp/last-check -type f | wc -l)
  last_commit=$(git -C tmp/backend-auth-oauth log -1 --format="%ar")
  
  if [[ $last_change -eq 0 && last_commit > 20 minutes ]]; then
    # Сигнал — нет активности
    osascript -e 'display notification "Агент не активен 20 мин" with title "loop-monitor"'
  fi
  
  sleep 60
done
```

Loop-monitor делает именно это — через Read и Bash(find, git log).

## Сигналы которые имеет смысл отслеживать

| Признак | Что значит |
|---|---|
| Нет изменений файлов N минут | Агент не работает или ждёт |
| Нет новых коммитов при активной задаче | Застрял или не коммитит прогресс |
| `.stop.log` не обновился | Сессия ещё жива (или упала без Stop-хука) |
| Размер `.jsonl` не растёт | Нет новых сообщений в сессии |

## Ограничение модели

Loop-monitor запущен на `haiku` — быстрая и дешёвая модель, подходит для polling. Не нужна умная модель для "проверь timestamp файла".

## Notifications Daemon (расширение)

Для production-мониторинга — отдельный Go-процесс на `fsnotify`:
- Следит за `~/.claude/projects/**/*.jsonl`
- При появлении нового `role: assistant` сообщения → агент ждёт ввода → macOS уведомление
- Не требует запущенного Claude

Статус: в разработке.

## Альтернатива: Stop-хук

Самый простой мониторинг — хук `Stop` в `settings.json`:

```bash
# modules/hooks/stop.sh
osascript -e 'display notification "Агент завершил задачу" with title "ai-workspace"'
```

Не watchdog, но покрывает самый частый кейс — "агент закончил, нужно забрать результат".
